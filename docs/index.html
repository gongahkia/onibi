Control agent work from your phone

Onibi routes terminal I/O and approval requests to a paired mobile browser over local HTTPS/WebSocket.

  • Approve, deny, or edit tool-call inputs.
  • Queue prompts while a session is busy.
  • Drive live sessions from the phone cockpit.

Host real local terminals

Run supported coding agents or wrap any local CLI/TUI while Onibi keeps state, audit records, and session routing local.

  • Claude Code, Codex, OpenCode, Goose, Gemini, Copilot CLI, Pi, and Amp adapters.
  • Opt-in shell command-done hooks for zsh, bash, and fish.
  • Local stdio MCP server for MCP-capable agents.

Use the mobile cockpit

The phone UI renders the live PTY with xterm.js, Ghostty defaults, and approval overlays when hooks block on a decision.

  • Top controls handle interrupt and kill; bottom soft keys handle Esc, Tab, arrows, control keys, paste, and theme.
  • Bundled terminfo, Ghostty themes, JetBrains Mono, Sixel/IIP rendering, and Kitty graphics transcoding keep the cockpit close to the Mac terminal.
  • Claude Code approvals can be approved, denied, or edited from the overlay.
  • Install the Onibi local CA profile once for iPhone HTTPS trust.

Single-user local daemon

The daemon uses local HTTPS/WebSocket for the phone cockpit, local Unix sockets with same-UID checks, OS secret storage, and SQLite state in the user profile.

  • No hosted account system or SaaS control plane.
  • Pair-by-QR owner setup with single-use tokens, doctor checks, and hook drift detection.
  • Built as a single Go binary for macOS and Linux.
