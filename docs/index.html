Onibi

Telegram-controlled coding-agent host for local terminals.

Run coding agents under local PTYs, approve tool calls from Telegram, queue prompts, inspect tails and terminal renders, and keep transport outbound-only.

Control agent work from Telegram

Onibi routes approval requests, turn-complete signals, prompt queues, PTY tails, and PNG previews to the owner chat.

  • Approve, deny, or edit tool-call inputs.
  • Queue prompts while a session is busy.
  • Target live sessions from Telegram commands or Mini App controls.

Host real local terminals

Run supported coding agents or wrap any local CLI/TUI while Onibi keeps state, audit records, and session routing local.

  • Claude Code, Codex, OpenCode, Goose, Gemini, Copilot CLI, Pi, and Amp adapters.
  • Opt-in shell command-done hooks for zsh, bash, and fish.
  • Local stdio MCP server for MCP-capable agents.

Choose plaintext or encrypted Telegram mode

Plain mode keeps reply-based Telegram UX. Encrypted mode sends AES-GCM envelopes through Telegram and decrypts inside the Mini App.

  • /secure opens encrypted prompt, interrupt, kill, and approval controls.
  • Self-host the static Mini App from an HTTPS origin you control.
  • Review the full model in the encrypted-mode and security docs.

Outbound-only, single-user daemon

The daemon uses outbound HTTPS to Telegram, local Unix sockets with same-UID checks, OS secret storage, and SQLite state in the user profile.

  • No inbound HTTP listener, webhook receiver, tunnel, account system, or SaaS control plane.
  • Pair-once owner setup with token rotation, doctor checks, and hook drift detection.
  • Built as a single Go binary for macOS and Linux.
